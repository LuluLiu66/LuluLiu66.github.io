1. navigation

DNS Lookup - resolving the web address, only at the first time of visiting), between Browser and DNS Server
TCP (Transmission Control Protocol) - between browser and the server which hold resource, via a three-way handshake(SYN=>SYN-ACK=>ACK)
TLS negotiation - for secure connections established over HTTPS, another handshake is needed. TLS negotiation determins which cipher will be used to encrypt the communication.
  • 1. Clienty says hello
  • 2. Server hello message and certificate
  • 3. Authentication
  • 4. The premaster secret
  • 5. Private kay used. The server decrypts the premaster secret
  • 6. Session keys created. From the client random, the server random, premaster secret
  • 7. Client finished.
  • 8. Server finished.
  • 9. Secure symmetric encryption achieved

2. data fetching

HTTP Request - an initial HTTP GET request, request for the HTML document, using HTTP protocol
The method - post, get, put, patch, delete etc
URI - stands for Uniform Resource Identifier. make up of 5 parts: scheme, authority, path, query, fragment
Request header fields - Respnse header fileds - HTTP Payload - HTTP Respnse - Status code - TCP Slow Start and congestion algorithms
"Slow Start" is an algorithm that balances the speed of a network connection. The first data packet will be 14kb (or smaller) and the way it works is that the amount of data transmitted is increased gradually until a predermined threshhold is reached.
"congestion algorithms" is since the connection has a limited capacity, if the server sends too many packets too quickly, they will be dropped, and the client won't send any ACK.
